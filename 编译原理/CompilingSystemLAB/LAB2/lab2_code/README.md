# LAB2 语义分析

本目录是实验二必做部分的代码。代码以实验一的词法分析和语法分析程序为基础，新增了手写语义分析模块。

## 文件说明

- `lexical.l`：Flex 词法分析规则。
- `syntax.y`：Bison 语法分析规则，并负责构造语法树。
- `treenode.h`、`treenode.c`：语法树结点结构和辅助函数。
- `semantic.h`、`semantic.c`：语义分析模块，检查必做的 1-17 类语义错误。
- `main.c`：程序入口，语法分析成功后调用语义分析。
- `Makefile`：Linux 环境下的编译和测试脚本。
- `testcases/required_*`：根据实验指导书必做样例整理的测试输入。
- `testcases/expected/*.txt`：每个测试输入对应的期望输出，用于自动比对。

## 编译方法

在本目录下执行：

```sh
make
```

编译成功后会生成可执行文件 `parser`。

## 运行方法

程序接收一个 C-- 源文件路径作为参数，例如：

```sh
./parser testcases/required_01
```

如果输入程序没有语义错误，程序不输出内容；如果存在语义错误，程序按实验要求输出错误类型、行号和说明文字。

## 测试方法

执行：

```sh
make test
```

该命令会依次运行 17 个必做样例，并将实际输出与 `testcases/expected` 中的期望输出进行比较。

## 实现范围

当前实现只覆盖实验二必做内容，即错误类型 1-17。函数声明、嵌套作用域、结构等价等选做要求没有实现。
