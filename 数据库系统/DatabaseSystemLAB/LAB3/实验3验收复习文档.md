# 实验3验收复习文档：System Manager 与 DDL

本文档根据 LAB3/lab3.pdf 和本次补齐的代码整理。目标是验收时能讲清楚 src/system 目录中数据库、表、列、索引元数据的组织方式，以及 open database、close database、drop table 的实现逻辑。

## 一、实验目标

Lab3 的核心任务是实现系统管理模块中与数据库打开、关闭、删表相关的 DDL 操作。

> 数据库目录中保存 db.meta、表文件和索引文件；System Manager 负责把这些磁盘文件和内存中的元数据、文件句柄对应起来。

本次按照指导书要求补齐：

- SmManager::open_db
- SmManager::close_db
- SmManager::drop_table

指导书当前实验没有要求补齐 create_index 和 drop_index，所以这两个函数仍保留为空，后续索引相关实验再实现。

## 二、核心概念

### 1. 数据库目录

在本项目中，一个数据库对应文件系统中的一个同名文件夹。例如：

~~~text
mydb/
  db.meta
  table1
  table2
  table1.index...
  LOG_FILE
~~~

SmManager::create_db 会创建目录并写入 db.meta。SmManager::open_db 会进入这个目录，读取元数据并打开表文件。

### 2. db.meta

db.meta 是数据库元数据文件，保存数据库名称、所有表的元数据、每张表的列信息和每张表上的索引信息。flush_meta() 会把内存中的 db_ 写回 db.meta。

### 3. DbMeta

DbMeta 表示一个数据库的元数据，核心成员是：

~~~cpp
std::string name_;
std::map<std::string, TabMeta> tabs_;
~~~

tabs_ 从表名映射到表元数据。判断表是否存在、获取表元数据都通过 DbMeta 完成。

### 4. TabMeta

TabMeta 表示一张表的元数据，核心成员包括：

- name：表名。
- cols：表的所有列元数据。
- indexes：表上已有索引的元数据。

### 5. ColMeta

ColMeta 表示一列的元数据，常见字段：

- tab_name：列属于哪张表。
- name：列名。
- type：列类型，如 int、float、string。
- len：列长度。
- offset：该列在一条记录中的偏移量。
- index：该列是否有索引。

验收回答：offset 很关键，因为执行器拿到一条记录的 char* data 后，就是通过 data + offset 找到某个字段的值。

### 6. fhs_ 和 ihs_

SmManager 运行时维护两个句柄表：

~~~cpp
std::unordered_map<std::string, std::unique_ptr<RmFileHandle>> fhs_;
std::unordered_map<std::string, std::unique_ptr<IxIndexHandle>> ihs_;
~~~

- fhs_：表名到记录文件句柄的映射。
- ihs_：索引文件名到索引句柄的映射。

验收回答：db_ 保存元数据，fhs_ 和 ihs_ 保存已经打开的文件句柄。SQL 执行时通过这些句柄读写表和索引。

## 三、src/system 文件夹整体讲解

### 1. sm_meta.h

这个文件定义数据库元数据结构，包括 ColMeta、IndexMeta、TabMeta、DbMeta。同时重载了 operator<< 和 operator>>，用于把元数据写入 db.meta 或从 db.meta 读回内存。

### 2. sm_manager.h

这是系统管理器的接口声明。DDL 语句最终会调用这里的方法，例如 create_db、drop_db、open_db、close_db、create_table、drop_table、show_tables、desc_table、create_index、drop_index。

### 3. sm_manager.cpp

这是系统管理器的具体实现。已经给出的代码包括创建数据库、删除数据库、创建表、显示表、描述表等。本次补齐的是打开数据库、关闭数据库、删除表。

### 4. 和其他模块的关系

System Manager 不直接管理页面内容，而是协调其他模块：调用 RmManager 创建、打开、关闭、删除表文件；调用 IxManager 打开、关闭、删除索引文件；为执行器提供 db_、fhs_、ihs_，让后续 DML 能找到表和索引。

/*
 * 验收讲解：System Manager 管理数据库级和表级元数据。
 * db_ 保存“有哪些表、表有哪些列、有哪些索引”等逻辑信息；fhs_ 和 ihs_ 保存已经打开的表文件/索引文件句柄。
 * open_db 要把磁盘上的 db.meta 恢复成内存元数据并打开文件句柄；close_db 要反过来写回元数据并关闭句柄；drop_table 要同步删除索引、表文件和元数据。
 */

/**
 * @description: 判断是否为一个文件夹
 * @return {bool} 返回是否为一个文件夹
 * @param {string&} db_name 数据库文件名称，与文件夹同名
 */
bool SmManager::is_dir(const std::string& db_name) {
    struct stat st;
    return stat(db_name.c_str(), &st) == 0 && S_ISDIR(st.st_mode);
}

/**
 * @description: 创建数据库，所有的数据库相关文件都放在数据库同名文件夹下
 * @param {string&} db_name 数据库名称
 */
void SmManager::create_db(const std::string& db_name) {
    if (is_dir(db_name)) {
        throw DatabaseExistsError(db_name);
    }
    //为数据库创建一个子目录
    std::string cmd = "mkdir " + db_name;
    if (system(cmd.c_str()) < 0) {  // 创建一个名为db_name的目录
        throw UnixError();
    }
    if (chdir(db_name.c_str()) < 0) {  // 进入名为db_name的目录
        throw UnixError();
    }
    //创建系统目录
    DbMeta *new_db = new DbMeta();
    new_db->name_ = db_name;

    // 注意，此处ofstream会在当前目录创建(如果没有此文件先创建)和打开一个名为DB_META_NAME的文件
    std::ofstream ofs(DB_META_NAME);

    // 将new_db中的信息，按照定义好的operator<<操作符，写入到ofs打开的DB_META_NAME文件中
    ofs << *new_db;  // 注意：此处重载了操作符<<

    delete new_db;

    // 创建日志文件
    disk_manager_->create_file(LOG_FILE_NAME);

    // 回到根目录
    if (chdir("..") < 0) {
        throw UnixError();
    }
}

/**
 * @description: 删除数据库，同时需要清空相关文件以及数据库同名文件夹
 * @param {string&} db_name 数据库名称，与文件夹同名
 */
void SmManager::drop_db(const std::string& db_name) {
    if (!is_dir(db_name)) {
        throw DatabaseNotFoundError(db_name);
    }
    std::string cmd = "rm -r " + db_name;
    if (system(cmd.c_str()) < 0) {
        throw UnixError();
    }
}

/**
 * @description: 打开数据库，找到数据库对应的文件夹，并加载数据库元数据和相关文件
 * @param {string&} db_name 数据库名称，与文件夹同名
 */
void SmManager::open_db(const std::string& db_name) {
    // 数据库在文件系统中表现为一个同名目录。
    // open_db 的第一步就是确认目录存在，并进入该目录，后续表文件、索引文件和 db.meta 都在此目录下访问。
    if (!is_dir(db_name)) {
        throw DatabaseNotFoundError(db_name);
    }
    if (chdir(db_name.c_str()) < 0) {
        throw UnixError();
    }

    // 重新打开数据库前清理内存中的旧句柄和旧元数据，避免上一次打开的数据库状态残留。
    db_ = DbMeta();
    fhs_.clear();
    ihs_.clear();

    // db.meta 保存 DbMeta 的序列化内容，operator>> 会恢复数据库名、表元数据、索引元数据。
    std::ifstream ifs(DB_META_NAME);
    if (!ifs.is_open()) {
        throw FileNotFoundError(DB_META_NAME);
    }
    ifs >> db_;

    // 打开每张表的数据文件，并把表名到 RmFileHandle 的映射记录在 fhs_ 中。
    // SQL 执行器后续通过 fhs_[tab_name] 访问表中的记录。
    for (auto &entry : db_.tabs_) {
        const std::string &tab_name = entry.first;
        TabMeta &tab = entry.second;
        fhs_.emplace(tab_name, rm_manager_->open_file(tab_name));

        // 如果表上存在索引，也要打开索引文件，并用索引文件名保存到 ihs_ 中。
        for (auto &index : tab.indexes) {
            std::string index_name = ix_manager_->get_index_name(tab_name, index.cols);
            ihs_.emplace(index_name, ix_manager_->open_index(tab_name, index.cols));
        }
    }
}

/**
 * @description: 把数据库相关的元数据刷入磁盘中
 */
void SmManager::flush_meta() {
    // 默认清空文件
    std::ofstream ofs(DB_META_NAME);
    ofs << db_;
}

/**
 * @description: 关闭数据库并把数据落盘
 */
void SmManager::close_db() {
    // 关闭数据库前先把最新元数据写回 db.meta，保证 create/drop table 后的状态能持久保存。
    flush_meta();

    // 关闭所有索引文件。IxManager::close_index 内部会写回索引文件头并 flush 对应 fd 的脏页。
    for (auto &entry : ihs_) {
        ix_manager_->close_index(entry.second.get());
    }
    ihs_.clear();

    // 关闭所有表数据文件。RmManager::close_file 内部会写回 file_hdr 并 flush 对应 fd 的脏页。
    for (auto &entry : fhs_) {
        rm_manager_->close_file(entry.second.get());
    }
    fhs_.clear();

    // open_db 进入了数据库目录；close_db 结束时回到上一级，方便之后打开/删除其他数据库。
    if (chdir("..") < 0) {
        throw UnixError();
    }
}

/**
 * @description: 显示所有的表,通过测试需要将其结果写入到output.txt,详情看题目文档
 * @param {Context*} context 
 */
void SmManager::show_tables(Context* context) {
    std::fstream outfile;
    outfile.open("output.txt", std::ios::out | std::ios::app);
    outfile << "| Tables |\n";
    RecordPrinter printer(1);
    printer.print_separator(context);
    printer.print_record({"Tables"}, context);
    printer.print_separator(context);

        auto &tab = entry.second;

        outfile << "| " << tab.name << " |\n";
    }
    printer.print_separator(context);
    outfile.close();
}

/**
 * @description: 显示表的元数据
 * @param {string&} tab_name 表名称
 * @param {Context*} context 
 */
void SmManager::desc_table(const std::string& tab_name, Context* context) {
    TabMeta &tab = db_.get_table(tab_name);

    std::vector<std::string> captions = {"Field", "Type", "Index"};
    RecordPrinter printer(captions.size());
    // Print header
    printer.print_separator(context);
    printer.print_record(captions, context);
    printer.print_separator(context);
    // Print fields
    for (auto &col : tab.cols) {
        std::vector<std::string> field_info = {col.name, coltype2str(col.type), col.index ? "YES" : "NO"};
        printer.print_record(field_info, context);
    }
    // Print footer
    printer.print_separator(context);
}

/**
 * @description: 创建表
 * @param {string&} tab_name 表的名称
 * @param {vector<ColDef>&} col_defs 表的字段
 * @param {Context*} context 
 */
void SmManager::create_table(const std::string& tab_name, const std::vector<ColDef>& col_defs, Context* context) {
    if (db_.is_table(tab_name)) {
        throw TableExistsError(tab_name);
    }
    // Create table meta
    int curr_offset = 0;
    TabMeta tab;
    tab.name = tab_name;
    for (auto &col_def : col_defs) {
        ColMeta col = {.tab_name = tab_name,
                       .name = col_def.name,
                       .type = col_def.type,
                       .len = col_def.len,
                       .offset = curr_offset,
                       .index = false};
        curr_offset += col_def.len;
        tab.cols.push_back(col);
    }
    // Create & open record file
    int record_size = curr_offset;  // record_size就是col meta所占的大小（表的元数据也是以记录的形式进行存储的）
    rm_manager_->create_file(tab_name, record_size);
    db_.tabs_[tab_name] = tab;
    // fhs_[tab_name] = rm_manager_->open_file(tab_name);
    fhs_.emplace(tab_name, rm_manager_->open_file(tab_name));

    flush_meta();
}

/**
 * @description: 删除表
 * @param {string&} tab_name 表的名称
 * @param {Context*} context
 */
void SmManager::drop_table(const std::string& tab_name, Context* context) {
    // 删表前必须先确认表存在，测试中 drop 不存在的表应抛 TableNotFoundError 并输出 failure。
    if (!db_.is_table(tab_name)) {
        throw TableNotFoundError(tab_name);
    }

    // 这里必须用引用。TabMeta 的拷贝构造只复制表名和列信息，不复制 indexes；如果用拷贝，下面就看不到该表已有索引。
    TabMeta &tab = db_.get_table(tab_name);

    // 先关闭并删除该表上的所有索引文件。
    // 文件必须先 close，才能 destroy，否则 DiskManager 会认为文件仍处于打开状态。
    for (auto &index : tab.indexes) {
        std::string index_name = ix_manager_->get_index_name(tab_name, index.cols);
        auto ih = ihs_.find(index_name);
        if (ih != ihs_.end()) {
            ix_manager_->close_index(ih->second.get());
            ihs_.erase(ih);
        }
        if (ix_manager_->exists(tab_name, index.cols)) {
            ix_manager_->destroy_index(tab_name, index.cols);
        }
    }

    // 关闭表数据文件并删除磁盘上的表文件。
    auto fh = fhs_.find(tab_name);
    if (fh != fhs_.end()) {
        rm_manager_->close_file(fh->second.get());
        fhs_.erase(fh);
    }
    rm_manager_->destroy_file(tab_name);

    // 最后更新内存中的数据库元数据，并刷回 db.meta。
    db_.tabs_.erase(tab_name);
    flush_meta();
}

## 五、代码实现逻辑详解

### 1. open_db

open_db 的作用是打开一个已经存在的数据库。流程是：判断数据库目录是否存在，进入数据库目录，清理旧的 db_、fhs_、ihs_，读取 db.meta，遍历表元数据打开每张表的数据文件，遍历索引元数据打开索引文件。

验收回答：open_db 不只是进入目录，还要把 db.meta 读到内存，并打开表文件和索引文件。否则后续 SQL 执行器没有句柄，无法读写表。

### 2. close_db

close_db 的作用是安全关闭当前数据库。流程是：flush_meta 写回元数据，关闭所有索引文件，关闭所有表数据文件，清空 ihs_ 和 fhs_，最后 chdir("..") 回到上级目录。

验收回答：关闭数据库前必须写回元数据并关闭所有文件句柄，否则建表、删表等修改可能没有持久化，打开的文件也可能影响后续删除或重新打开。

### 3. drop_table

drop_table 的作用是删除一张表及其相关文件。流程是：检查表存在，取得 TabMeta 引用，先关闭并删除该表上的索引文件，再关闭并删除表文件，最后从 db_.tabs_ 中移除表元数据并 flush_meta。

这里必须使用 TabMeta &tab 引用。因为 TabMeta 的拷贝构造只复制表名和列信息，不复制 indexes；如果写成拷贝，删表时可能看不到该表上的索引。

## 六、常见验收问题和答案

### 1. db.meta 有什么作用？

它持久化保存数据库有哪些表、每张表有哪些列、列的类型和偏移、有哪些索引。关闭数据库或修改 DDL 后需要写回它，打开数据库时再读回来。

### 2. 为什么 open_db 要 chdir(db_name)？

因为表文件、索引文件、db.meta 都存放在数据库目录内部。进入目录后，后续用表名打开文件就能直接定位到当前数据库的文件。

### 3. fhs_ 是什么？

fhs_ 是表名到 RmFileHandle 的映射。执行插入、删除、更新、查询时，都需要通过 fhs_[tab_name] 找到表文件句柄。

### 4. ihs_ 是什么？

ihs_ 是索引文件名到 IxIndexHandle 的映射。打开数据库时把已有索引打开，后续如果执行器需要使用索引，就能通过这个映射找到索引句柄。

### 5. create_table 中 offset 是怎么计算的？

从 0 开始累计每列长度。这样一条记录中的每列都能通过 data + offset 直接访问。
