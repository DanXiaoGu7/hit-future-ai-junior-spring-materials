# 实验2验收复习文档：Record Manager

本文档根据 LAB2/lab2.pdf 和本次补齐的代码整理。目标是验收时能讲清楚 src/record 目录的整体设计、我们补齐的代码逻辑，以及常见追问的回答。

## 一、实验目标

Lab2 的核心任务是实现记录管理器。可以用一句话概括：

> 在文件页之上管理一张表中的定长记录，支持按 Rid 取记录、插入记录、删除记录、更新记录，以及顺序扫描所有有效记录。

Record Manager 建立在 Lab1 的 BufferPoolManager 之上。它不直接读写磁盘，而是通过缓冲池拿到页面，再在页面内部维护记录槽位。

## 二、核心概念

### 1. Rid

Rid 是一条记录的位置编号，由两部分组成：

~~~cpp
int page_no;
int slot_no;
~~~

page_no 表示记录在哪个数据页，slot_no 表示记录在该页的第几个槽位。Rid 是数据库内部的逻辑地址，不是内存地址。

### 2. RmFileHdr

RmFileHdr 是记录文件头，保存在文件第 0 页，记录整张表的布局信息：

- record_size：每条记录的固定长度。
- num_pages：文件目前一共有多少页。
- num_records_per_page：每个数据页最多放多少条记录。
- first_free_page_no：空闲页链表的头结点页号。
- bitmap_size：每个数据页 bitmap 占多少字节。

验收回答：文件头负责描述整个记录文件，尤其是告诉我们一页能放多少条记录，以及哪里还有空槽可以插入。

### 3. RmPageHdr

每个数据页开头都有页头，记录该页自己的状态：

- num_records：当前页已经存放多少条记录。
- next_free_page_no：如果当前页还有空槽，它在空闲页链表中指向下一页。

### 4. bitmap

每个数据页都有 bitmap。bitmap 的每一位对应一个 slot：

~~~text
bitmap bit = 1：该 slot 已经有记录
bitmap bit = 0：该 slot 为空，可以插入
~~~

删除记录时只需要把 bitmap 对应位清 0，不一定要清空 slot 里的旧字节，因为系统判断记录是否有效以 bitmap 为准。

### 5. slot

slot 是页内真正存储记录数据的位置。因为本实验是定长记录，所以可以通过下面的公式直接计算某个槽的起始地址：

~~~cpp
slots + slot_no * record_size
~~~

### 6. 空闲页链表

file_hdr_.first_free_page_no 指向第一张还有空槽的数据页，每个空闲页通过 page_hdr->next_free_page_no 串起来。

- 插入记录时，从空闲页链表头找页。
- 页面插满后，从空闲页链表移除。
- 满页删除一条记录后，重新加入空闲页链表。

验收回答：空闲页链表的作用是快速找到还能插入记录的页面，避免每次插入都从第一页扫描到最后一页。

## 三、src/record 文件夹整体讲解

### 1. rm_defs.h

这个文件定义记录管理模块的基础结构。

重点结构包括：

- Rid：记录位置。
- RmFileHdr：记录文件头。
- RmPageHdr：记录页页头。
- RmRecord：一条记录在内存中的拷贝。

RmRecord 内部有 char *data，它保存一条记录的实际字节内容。因为表中的字段可能有 int、float、char(n)，底层统一按字节数组保存。

### 2. rm_manager.h 和 rm_manager.cpp

RmManager 负责记录文件级别的操作：

- create_file：创建一张表对应的记录文件，并初始化第 0 页文件头。
- destroy_file：删除记录文件。
- open_file：打开记录文件，读取文件头，返回 RmFileHandle。
- close_file：写回文件头并关闭文件。

它不负责具体某条记录的插入删除，具体记录操作交给 RmFileHandle。

### 3. rm_file_handle.h 和 rm_file_handle.cpp

RmFileHandle 表示已经打开的一张表文件。它是 Lab2 最重要的类。

它掌握：

- fd_：底层文件描述符。
- file_hdr_：记录文件头。
- disk_manager_：磁盘管理器。
- buffer_pool_manager_：缓冲池管理器。

它负责通过 Rid 读取记录、自动找空槽插入记录、删除记录并维护 bitmap、更新记录内容，以及创建和释放记录页。

### 4. rm_scan.h 和 rm_scan.cpp

RmScan 是表扫描器，用来顺序遍历一张表中的所有有效记录。它不读取整张表到内存，而是保存当前 rid_，每次 next() 根据 bitmap 找下一条有效记录。

### 5. bitmap.h

bitmap.h 提供位图工具函数，例如 Bitmap::is_set、Bitmap::set、Bitmap::reset、Bitmap::first_bit、Bitmap::next_bit。Record Manager 用 bitmap 判断页内哪些 slot 有记录，哪些 slot 为空。

/*
 * 验收讲解：本文件实现“表文件中一条条记录怎么放、怎么找、怎么改”。
 * 记录文件的第 0 页是文件头，真实记录页从 1 号页开始；每个记录页内部又分成 page header、bitmap、slots 三部分。
 * bitmap 决定 slot 是否有效，slot 区域保存记录的原始字节；所以增删查改的本质都是维护 bitmap、页头记录数和 slot 字节。
 * 只要通过 BufferPoolManager 拿到页面，该页面就是 pin 状态，用完必须 unpin；读操作 dirty=false，修改 bitmap/页头/slot 后 dirty=true。
 */

/**
 * @description: 获取当前表中记录号为rid的记录
 * @param {Rid&} rid 记录号，指定记录的位置
 * @param {Context*} context
 * @return {unique_ptr<RmRecord>} rid对应的记录对象指针
 */
std::unique_ptr<RmRecord> RmFileHandle::get_record(const Rid& rid, Context* context) const {
    // Rid 由 page_no 和 slot_no 组成：page_no 定位记录所在的数据页，slot_no 定位页内槽位。
    // fetch_page_handle 会把目标页 pin 到缓冲池中，并返回 RmPageHandle 方便访问页头、bitmap 和 slots。
    RmPageHandle page_handle = fetch_page_handle(rid.page_no);

    // bitmap 中 1 表示该 slot 有记录，0 表示空槽。
    // 如果 slot 越界或 bitmap 位为 0，说明 rid 指向的位置没有有效记录。
    if (rid.slot_no < 0 || rid.slot_no >= file_hdr_.num_records_per_page ||
        !Bitmap::is_set(page_handle.bitmap, rid.slot_no)) {
        buffer_pool_manager_->unpin_page(page_handle.page->get_page_id(), false);
        throw RecordNotFoundError(rid.page_no, rid.slot_no);
    }

    // RmRecord(size, data) 会分配独立内存并拷贝槽位数据。
    // 因此即使后续页面 unpin 或被淘汰，返回的记录内容仍然有效。
    auto record = std::make_unique<RmRecord>(file_hdr_.record_size, page_handle.get_slot(rid.slot_no));

    // get_record 只读页面，不修改任何页内内容，所以 dirty=false。
    buffer_pool_manager_->unpin_page(page_handle.page->get_page_id(), false);
    return record;
}

/**
 * @description: 在当前表中插入一条记录，不指定插入位置
 * @param {char*} buf 要插入的记录的数据
 * @param {Context*} context
 * @return {Rid} 插入的记录的记录号（位置）
 */
Rid RmFileHandle::insert_record(char* buf, Context* context) {
    // 获取一个还有空槽的数据页。若当前没有空闲页，create_page_handle 会创建新页。
    RmPageHandle page_handle = create_page_handle();

    // 在 bitmap 中查找第一个 0 位，0 位代表对应 slot 为空，可以放新记录。
    int slot_no = Bitmap::first_bit(false, page_handle.bitmap, file_hdr_.num_records_per_page);
    if (slot_no == file_hdr_.num_records_per_page) {
        buffer_pool_manager_->unpin_page(page_handle.page->get_page_id(), false);
        throw InternalError("RmFileHandle::insert_record: no free slot in free page");
    }

    // 将定长记录拷贝到空槽，并把 bitmap 对应位置置 1，表示该槽已有记录。
    memcpy(page_handle.get_slot(slot_no), buf, file_hdr_.record_size);
    Bitmap::set(page_handle.bitmap, slot_no);
    page_handle.page_hdr->num_records++;

    // Rid 只记录位置，不保存数据内容；以后 get/update/delete 都靠这个 page_no + slot_no 再定位到记录。
    Rid rid{page_handle.page->get_page_id().page_no, slot_no};

    // 如果插入后页面变满，它不再有空槽，必须从文件头维护的空闲页链表中移除。
    // 因为 create_page_handle 总是返回 first_free_page_no 指向的页，所以这里更新链表头即可。
    if (page_handle.page_hdr->num_records == file_hdr_.num_records_per_page) {
        file_hdr_.first_free_page_no = page_handle.page_hdr->next_free_page_no;
        page_handle.page_hdr->next_free_page_no = RM_NO_PAGE;
    }

    // 插入修改了页面数据、页头和 bitmap，所以 dirty=true。
    buffer_pool_manager_->unpin_page(page_handle.page->get_page_id(), true);
    return rid;
}

/**
 * @description: 在当前表中的指定位置插入一条记录
 * @param {Rid&} rid 要插入记录的位置
 * @param {char*} buf 要插入记录的数据
 */
void RmFileHandle::insert_record(const Rid& rid, char* buf) {
    // 指定位置插入主要供恢复或后续模块使用；普通插入由 insert_record(buf, context) 自动找空槽。
    RmPageHandle page_handle = fetch_page_handle(rid.page_no);
    if (rid.slot_no < 0 || rid.slot_no >= file_hdr_.num_records_per_page) {
        buffer_pool_manager_->unpin_page(page_handle.page->get_page_id(), false);
        throw RecordNotFoundError(rid.page_no, rid.slot_no);
    }

    bool was_empty = !Bitmap::is_set(page_handle.bitmap, rid.slot_no);
    if (was_empty) {
        Bitmap::set(page_handle.bitmap, rid.slot_no);
        page_handle.page_hdr->num_records++;
    }
    memcpy(page_handle.get_slot(rid.slot_no), buf, file_hdr_.record_size);

    // 如果指定插入使页面正好变满，也需要从空闲页链表移除。
    // 普通 insert_record 总是使用链表头页；指定 rid 插入则可能命中链表中间的页，所以这里按单链表方式查找前驱。
    if (was_empty && page_handle.page_hdr->num_records == file_hdr_.num_records_per_page) {
        if (file_hdr_.first_free_page_no == rid.page_no) {
            file_hdr_.first_free_page_no = page_handle.page_hdr->next_free_page_no;
        } else {
            int prev_page_no = file_hdr_.first_free_page_no;
            while (prev_page_no != RM_NO_PAGE) {
                RmPageHandle prev_handle = fetch_page_handle(prev_page_no);
                if (prev_handle.page_hdr->next_free_page_no == rid.page_no) {
                    prev_handle.page_hdr->next_free_page_no = page_handle.page_hdr->next_free_page_no;
                    buffer_pool_manager_->unpin_page(prev_handle.page->get_page_id(), true);
                    break;
                }
                int next_page_no = prev_handle.page_hdr->next_free_page_no;
                buffer_pool_manager_->unpin_page(prev_handle.page->get_page_id(), false);
                prev_page_no = next_page_no;
            }
        }
        page_handle.page_hdr->next_free_page_no = RM_NO_PAGE;
    }

    buffer_pool_manager_->unpin_page(page_handle.page->get_page_id(), true);
}

/**
 * @description: 删除记录文件中记录号为rid的记录
 * @param {Rid&} rid 要删除的记录的记录号（位置）
 * @param {Context*} context
 */
void RmFileHandle::delete_record(const Rid& rid, Context* context) {
    RmPageHandle page_handle = fetch_page_handle(rid.page_no);
    if (rid.slot_no < 0 || rid.slot_no >= file_hdr_.num_records_per_page ||
        !Bitmap::is_set(page_handle.bitmap, rid.slot_no)) {
        buffer_pool_manager_->unpin_page(page_handle.page->get_page_id(), false);
        throw RecordNotFoundError(rid.page_no, rid.slot_no);
    }

    // 如果删除前页面是满的，那么删除后它会从“无空槽”变成“有空槽”，需要重新挂回空闲页链表。
    bool was_full = page_handle.page_hdr->num_records == file_hdr_.num_records_per_page;

    Bitmap::reset(page_handle.bitmap, rid.slot_no);
    page_handle.page_hdr->num_records--;

    if (was_full) {
        release_page_handle(page_handle);
    }

    buffer_pool_manager_->unpin_page(page_handle.page->get_page_id(), true);
}


/**
 * @description: 更新记录文件中记录号为rid的记录
 * @param {Rid&} rid 要更新的记录的记录号（位置）
 * @param {char*} buf 新记录的数据
 * @param {Context*} context
 */
void RmFileHandle::update_record(const Rid& rid, char* buf, Context* context) {
    RmPageHandle page_handle = fetch_page_handle(rid.page_no);
    if (rid.slot_no < 0 || rid.slot_no >= file_hdr_.num_records_per_page ||
        !Bitmap::is_set(page_handle.bitmap, rid.slot_no)) {
        buffer_pool_manager_->unpin_page(page_handle.page->get_page_id(), false);
        throw RecordNotFoundError(rid.page_no, rid.slot_no);
    }

    // 本实验采用定长记录，更新时直接覆盖原槽位即可，不需要移动其他记录。
    memcpy(page_handle.get_slot(rid.slot_no), buf, file_hdr_.record_size);
    buffer_pool_manager_->unpin_page(page_handle.page->get_page_id(), true);
}

/**
 * @description: 获取指定页面的页面句柄
 * @param {int} page_no 页面号
 * @return {RmPageHandle} 指定页面的句柄
 */
RmPageHandle RmFileHandle::fetch_page_handle(int page_no) const {
    // 0 号页保存文件头，真实记录页从 1 号页开始；page_no >= num_pages 表示页面尚未分配。
    if (page_no < RM_FIRST_RECORD_PAGE || page_no >= file_hdr_.num_pages) {
        throw PageNotExistError(disk_manager_->get_file_name(fd_), page_no);
    }

    PageId page_id{fd_, page_no};
    Page *page = buffer_pool_manager_->fetch_page(page_id);
    if (page == nullptr) {
        throw InternalError("RmFileHandle::fetch_page_handle: failed to fetch page");
    }
    return RmPageHandle(&file_hdr_, page);
}

/**
 * @description: 创建一个新的page handle
 * @return {RmPageHandle} 新的PageHandle
 */
RmPageHandle RmFileHandle::create_new_page_handle() {
    PageId page_id{fd_, INVALID_PAGE_ID};
    Page *page = buffer_pool_manager_->new_page(&page_id);
    if (page == nullptr) {
        throw InternalError("RmFileHandle::create_new_page_handle: failed to create page");
    }

    RmPageHandle page_handle(&file_hdr_, page);

    // 初始化新记录页：记录数为 0；bitmap 全 0；新页有空槽，因此要加入空闲页链表。
    page_handle.page_hdr->num_records = 0;
    page_handle.page_hdr->next_free_page_no = file_hdr_.first_free_page_no;
    Bitmap::init(page_handle.bitmap, file_hdr_.bitmap_size);

    // new_page 分配了新的 page_no，文件总页数加 1，并把新页设为空闲页链表头。
    file_hdr_.num_pages++;
    file_hdr_.first_free_page_no = page_id.page_no;

    return page_handle;
}

/**
 * @brief 创建或获取一个空闲的page handle
 *
 * @return RmPageHandle 返回生成的空闲page handle
 * @note pin the page, remember to unpin it outside!
 */
RmPageHandle RmFileHandle::create_page_handle() {
    // first_free_page_no 指向第一个仍有空槽的数据页。
    // 如果没有空闲页，就创建新页；否则直接取出链表头对应的页。
    if (file_hdr_.first_free_page_no == RM_NO_PAGE) {
        return create_new_page_handle();
    }
    return fetch_page_handle(file_hdr_.first_free_page_no);
}

/**
 * @description: 当一个页面从没有空闲空间的状态变为有空闲空间状态时，更新文件头和页头中空闲页面相关的元数据
 */
void RmFileHandle::release_page_handle(RmPageHandle&page_handle) {
    // 满页原本不在空闲页链表中；删除记录后它有了空槽，需要插到空闲页链表头部。
    page_handle.page_hdr->next_free_page_no = file_hdr_.first_free_page_no;
    file_hdr_.first_free_page_no = page_handle.page->get_page_id().page_no;
}

#include "rm_scan.h"
#include "rm_file_handle.h"

/*
 * 验收讲解：RmScan 是记录文件的顺序扫描器。
 * 它不把整张表读入内存，只保存当前 Rid；每次 next() 都根据页内 bitmap 寻找下一条有效记录。
 * 因为删除记录只会把 bitmap 清 0，旧 slot 字节可能还在，所以扫描时必须以 bitmap 为准，不能直接遍历 slot 数据。
 */

/**
 * @brief 初始化file_handle和rid
 * @param file_handle
 */
RmScan::RmScan(const RmFileHandle *file_handle) : file_handle_(file_handle) {
    // RmScan 用 rid_ 表示当前扫描器指向的记录位置。
    // 初始化时先放在“第一张记录页的前一个槽位”，再调用 next() 寻找第一条真实记录。
    rid_ = {RM_FIRST_RECORD_PAGE, -1};
    next();
}

/**
 * @brief 找到文件中下一个存放了记录的位置
 */
void RmScan::next() {
    // 从当前 rid_ 的下一个 slot 开始找 bitmap 为 1 的位置。
    // bitmap 为 1 表示该槽中存放了有效记录。
    while (rid_.page_no < file_handle_->file_hdr_.num_pages) {
        RmPageHandle page_handle = file_handle_->fetch_page_handle(rid_.page_no);
        // next_bit 的第四个参数是当前 slot；函数会从它后面继续找，避免重复返回同一条记录。
        int next_slot = Bitmap::next_bit(
            true, 
            page_handle.bitmap,
            file_handle_->file_hdr_.num_records_per_page, rid_.slot_no
        );
        file_handle_->buffer_pool_manager_->unpin_page(page_handle.page->get_page_id(), false);

        if (next_slot < file_handle_->file_hdr_.num_records_per_page) {
            rid_.slot_no = next_slot;
            return;
        }

        // 当前页后面没有更多记录，移动到下一页，并从 -1 开始搜索该页第一个记录。
        rid_.page_no++;
        rid_.slot_no = -1;
    }

    // page_no 到达 num_pages 表示已经越过最后一页，把它作为扫描结束标记。
    rid_ = {RM_NO_PAGE, -1};
}

/**
 * @brief ​ 判断是否到达文件末尾
 */
bool RmScan::is_end() const {
    return rid_.page_no == RM_NO_PAGE;
}

/**
 * @brief RmScan内部存放的rid
 */
Rid RmScan::rid() const {
    return rid_;
}

## 五、代码实现逻辑详解

### 1. get_record

流程如下：根据 rid.page_no 获取目标页，检查 slot_no 是否越界，检查 bitmap 对应位是否为 1。如果该位置没有记录，抛 RecordNotFoundError。如果存在记录，就把 slot 中的定长字节拷贝成 RmRecord。因为只是读取，所以 unpin_page 时 dirty=false。

验收回答：get_record 不能直接相信 Rid，一定要查 bitmap。只有 bitmap 为 1，才说明这个 slot 里真的有记录。

### 2. insert_record(char *buf)

流程如下：调用 create_page_handle 找到有空槽的页，用 Bitmap::first_bit 找第一个空 slot，把记录拷贝进去，设置 bitmap，num_records 加一。如果页面插满，就从空闲页链表中移除。因为修改了页面内容、页头和 bitmap，所以 dirty=true。

### 3. insert_record(const Rid&, char*)

这是指定位置插入。普通插入自动找空槽，这个函数主要给恢复模块或特殊场景使用。如果指定插入让页面变满，就从空闲页链表中移除。因为指定 Rid 可能命中链表中间的页，所以代码会沿链表查找前驱页。

### 4. delete_record

流程如下：取出目标页，检查 slot 合法且 bitmap 为 1，记录删除前页面是否满页，然后把 bitmap 清 0，num_records 减一。如果删除前是满页，删除后该页第一次出现空槽，需要调用 release_page_handle 加回空闲页链表。

验收回答：删除记录不需要真的清空 slot 内存，只要把 bitmap 清 0。因为系统判断记录是否存在只看 bitmap。

### 5. update_record

通过 Rid 定位页面和 slot，确认记录存在，然后用新记录字节覆盖旧 slot。因为本实验是定长记录，更新不会改变记录大小，也不需要移动其他记录。

### 6. fetch_page_handle

0 号页保存文件头，真实记录页从 1 号页开始。fetch_page_handle 先检查 page_no 是否合法，再通过 BufferPoolManager 获取页面，并包装成 RmPageHandle。

### 7. create_new_page_handle

新建记录页时要初始化页头、bitmap 和文件头。新页一定是空闲页，所以创建后要挂到空闲页链表头。

### 8. create_page_handle

如果 first_free_page_no == RM_NO_PAGE，说明没有空闲页，创建新页；否则直接取空闲链表头对应的页。

### 9. release_page_handle

当一个满页删除记录后，它重新有了空槽，所以要通过头插法放回空闲页链表。

### 10. RmScan

RmScan 构造时把 rid_ 初始化到第一张记录页的前一个槽位，然后调用 next() 找第一条有效记录。next() 在当前页找 bitmap 为 1 的下一个 slot，当前页找不到就移动到下一页。扫描结束后 rid_ 设为 RM_NO_PAGE。

## 六、常见验收问题和答案

### 1. 为什么记录页要用 bitmap？

因为删除记录后页内会出现空洞。bitmap 可以用很小的空间标记每个 slot 的使用状态。

### 2. 为什么删除记录只清 bitmap，不清空数据？

因为数据是否有效由 bitmap 决定。只要 bitmap 为 0，这个 slot 就被认为没有记录，旧字节会在下次插入时被覆盖。

### 3. 为什么插入后页面满了要从空闲页链表移除？

空闲页链表只应该保存还有空槽的页。满页没有插入空间，如果不移除，下次插入可能找不到空 slot。

### 4. 为什么删除前要判断页面是否满？

只有满页原来不在空闲页链表中。删除后它从满页变成有空槽，才需要重新加入空闲页链表。如果页面本来就有空槽，它已经在链表里，不能重复加入。

### 5. 为什么获取记录后要 unpin？

fetch_page_handle 会通过缓冲池 pin 页面，表示正在使用。用完后必须 unpin，否则 pin_count 无法降到 0，页面永远不能被替换。

### 6. dirty=true 和 dirty=false 怎么判断？

只读操作用 false，修改页面内容、页头或 bitmap 的操作用 true。例如 get_record 是 false，插入、删除、更新都是 true。

### 7. 一条记录实际存在哪里？

实际存储在记录页的 slot 区域里。内存中访问时，通过 RmPageHandle::get_slot(slot_no) 得到对应 slot 的 char* 指针。
